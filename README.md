# Arx

Architecture guardrails for codebases built by humans and AI agents.

Arx enforces dependency direction rules against a declared blueprint. It catches architecture violations before they ship — in your terminal, in CI, or as real-time feedback to AI coding tools like Claude Code.

```bash
arx infer --repo .                          # generate a blueprint from your code
arx scan  --repo . --blueprint arch.blu     # see what's wrong
arx check --repo . --blueprint arch.blu     # block PRs that violate the blueprint
```

Two commands from zero to value. One more to gate your pipeline.

---

## Why

Codebases decay. A developer under deadline shortcuts from adapter straight to domain. An AI agent wires a database call into the business logic because it optimizes for "works" not "works and stays maintainable."

Arx makes the intended architecture executable. You declare the rules once in a `.blu` file, and arx enforces them continuously — whether the code is written by a person or generated by an AI.

## Install

Download the binary for your platform from [GitHub Releases](https://github.com/WattWozy/arx/releases/latest):

```bash
# Linux / macOS
chmod +x arx-linux-amd64
mv arx-linux-amd64 /usr/local/bin/arx
```

Or run via JAR (requires Java 21+): `java -jar arx.jar`

---

## The blueprint

A plain-text file declaring your intended architecture:

```
module domain      dev.myapp.domain.**      layer=0
module application dev.myapp.application.** layer=1
module adapter     dev.myapp.adapter.**     layer=2

allow application -> domain
allow adapter     -> application
allow adapter     -> domain
```

Everything not in an `allow` rule is a violation. Generate a starting point with `arx infer`, then tighten it to match your intent.

## Core commands

| Command | What it does |
|---------|-------------|
| `arx infer` | Generates a blueprint from your package structure |
| `arx scan` | Full analysis: violations, metrics, cycles, hotspots, trends |
| `arx check` | CI gate — silent on pass, exit 1 on failure |
| `arx watch` | Real-time feedback on file save or changed-file list |
| `arx query` | Ask natural language questions about your architecture |

## CI integration

```yaml
- name: Architecture check
  run: |
    arx check \
      --repo . \
      --blueprint arch.blu \
      --fail-on new-violations \
      --fail-on new-cycles
```

Fail conditions: `any-violations`, `new-violations`, `new-cycles`, `stale-blueprint`, `instability-threshold=<N>`.

## AI agent integration (MCP)

Arx runs as an [MCP server](https://modelcontextprotocol.io), giving Claude Code and other AI tools direct access to architecture analysis — no shell parsing, just structured JSON.

Add to `.claude/settings.json`:

```json
{
  "mcpServers": {
    "arx": {
      "command": "arx",
      "args": ["mcp-serve"]
    }
  }
}
```

The AI agent can then check violations, get metrics, infer blueprints, and query architecture health as part of its coding loop. When it introduces a violation, it gets structured feedback and self-corrects before you see the PR.

---

## What it measures

Arx tracks per-module: instability, abstractness, fan-in/out, hub score, hotspots, churn acceleration, bus factor risk, PageRank, betweenness centrality, CRAP score (with coverage data), and test debt. Violations persisting 3+ snapshots are flagged as chronic. Cycles are detected via Tarjan's SCC. Trend direction (improving / stable / degrading) is computed across commit history.

Full metrics reference, blueprint syntax, output formats, environment variables, and advanced configuration are in [docs.md](docs.md).
